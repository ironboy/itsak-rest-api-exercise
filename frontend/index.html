Solution to the exercise "Create a REST api"

This exercise is about creating a REST api for a database with the tables pets and petOwners and the views petOwnersAndPets, petsWithoutOwners and peopleWithoutPets.

So there isn't much of a frontend! All code important to the solution is in the backend.js-file.

Get routes to try out

  • /api/pets
  • /api/pets/1 (try with other ids than 1 too)
  • /api/petOwners
  • /api/petOwners/2 (try with other ids than 2 too)
  • /api/petOwnersAndPets
  • /api/petOwnersAndPets/4 (try with other ids than 4 too)
  • /api/petsWithoutOwners
  • /api/petsWithoutOwners/5 (try with other ids than 5 too)
  • /api/peopleWithoutPets
  • /api/peopleWithoutPets/6 (try with other ids than 6 too)

Post, put and delete routes to try out

There are POST, PUT and DELETE routes for the tables (but not for the views). Those you can't try out directly in your browser, instead you need a tool like Thunder Client.
